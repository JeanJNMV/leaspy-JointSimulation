These peaks are mainly a mechanical effect of the current joint simulation pipeline: event times are sampled independently, and when an event occurs very early, all later visits are removed, so some patients end up with only their baseline visit. For those patients, follow-up duration is exactly zero by construction because it is computed as last time minus first time, and with one visit this difference is 0. In addition, when an event is simulated before the first planned visit, the algorithm keeps a single first visit and censors the event, which further increases the mass near one visit and near zero follow-up. So the phenomenon does not necessarily mean the model is globally wrong, but it does indicate a mismatch in visit-process realism if the original cohort has far fewer such short trajectories.
